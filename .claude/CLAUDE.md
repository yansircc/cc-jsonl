# cc-jsonl

Claude Code 对话记录搜索工具。

## 设计哲学

```
V = C / F
```

**Value = Content / Friction**

Content 已满（JSONL 记录了一切），最大化价值 ≡ 最小化摩擦。
每个设计决策归约为一个判据：**这是否降低了 F？**

### 三条不可约原则

1. **镜头，不是容器** — 不存储、不转换、不整理，只聚焦。JSONL 是神圣的原始记录，零冗余信息。
2. **查询即组织** — 没有分类、标签、文件夹。用户的搜索词本身就是结构。
3. **说过一次 → 找得到** — 唯一的功能契约。

### 反向验证

偏离公式的信号：
- 建数据库 → 在造容器（违反 1）
- 加标签/分类 → 在替用户组织（违反 2）
- 加摘要/预览以外的 AI 功能 → 在增加 C 而不是降低 F

## 技术栈

- 后端：Go（标准库 HTTP server，无框架）
- 前端：SvelteKit + SASS
- 风格：Craigslist 极简
- 数据源：~/.claude/projects/ 下的 JSONL 文件，只读，不建数据库

## 开发规范

- 所有代码注释和 commit message 使用中文
- Go 代码遵循标准 Go 项目布局
- 前端代码使用 TypeScript
